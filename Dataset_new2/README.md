# 意图识别数据集——轨迹预测版（Intent Recognition Dataset — Trajectory Prediction Variant）

## 概述

本数据集面向**航天器相对运动轨迹预测**任务，也可用于意图识别中的行为建模。与主数据集（`Dataset/`）不同，本版本**不含递推预测轨迹和显式标签**，而是将轨迹截取/补齐后的序列直接按时间切分：前 10 步为观测输入（`X_now`），后 10 步为未来真值（`X_next`）。数据同样源于 PPO 强化学习训练的追踪航天器对非机动目标逼近的仿真轨迹。

---

## 文件清单与角色

| 文件 | 角色 | 说明 |
|------|------|------|
| `X_now.csv` | 输入特征 | 追踪航天器相对于目标（位于 LVLH 系原点）的近期 10 个时间步的相对状态 |
| `X_next.csv` | 预测目标 | 紧接 `X_now` 之后的下 10 个时间步的相对状态（可能包含 CW 外推补齐点，详见下文） |

> **对应关系**：两个文件按行严格一一对应。第 *i* 行 `X_now` 与第 *i* 行 `X_next` 取自同一批次的同一段轨迹。**绝对不可单独打乱任一文件的顺序。**

---

## 与主数据集（`Dataset/`）的关键区别

| 方面 | 主数据集（Dataset/） | 本数据集（Dataset_new2/） |
|------|---------------------|--------------------------|
| 输入 | `X_real`：真实轨迹前 10 步 | `X_now`：截取/补齐后序列的前 10 步（等价） |
| 输出/目标 | `X_recurrence`：以 X_real 终点为初值，用 CW 矩阵 **单独递推** 10 步 | `X_next`：序列的第 11~20 步，**直接从同一个截取/补齐序列中取后半段** |
| 标签 | `Y.csv`：包含 N、min_distance、phi | 无独立标签文件 |
| 任务类型 | 意图识别（含几何特征标签） | 轨迹预测 / 时序预测（端到端） |

> **核心差异**：`X_next` 不是单独递推生成的——它来自与 `X_now` 同一段截取/补齐后的序列。对于原始轨迹足够长的部分，`X_next` 就是真实仿真数据；对于原轨迹长度不足需要外推补齐的部分，`X_next` 中会混入 CW 外推点。这使 `X_next` 比 `X_recurrence` 更接近"真实未来轨迹"（因为 X_recurrence 全部都是 CW 递推值，而 X_next 只有必要时才含外推值）。

## 数据来源与采集

### 仿真实验

- **任务场景**：近地圆轨道上，一颗追踪航天器接近一颗非机动目标（目标位于 LVLH 坐标系原点）。追踪航天器通过 PPO 策略输出速度增量进行机动，目标是使相对距离降至 5 km 以内。
- **成功判定**：相对距离 ≤ 5 km → 拦截成功，回合结束，整条轨迹存入 `.mat` 文件。
- **失败判定**：相对距离 ≥ 200 km 或达到最大步数（100 步）→ 回合失败，轨迹不保存。
- **数据采集**：仅保留成功回合的完整轨迹数据，保存至 `Success_Data/` 目录（.mat 文件），随后手动复制到 MATLAB 批处理脚本的输入文件夹 `Capture_T60h1/`。

### 物理环境参数

| 参数 | 符号 | 值 | 单位 |
|------|------|-----|------|
| 轨道半径 | r | 6371 + 480 = 6851 | km |
| 地球引力常数 | μ | 398600 | km³/s² |
| 轨道平均角速度 | n | sqrt(μ/r³) ≈ 0.001134 | rad/s |
| RL 仿真时间步长 | h | 1 | s |
| CW 外推步长 | T | 60 | s |
| 追踪航天器初始距离范围 | — | 100 ~ 120 | km |
| 追踪航天器初始俯仰角 | — | π/2 ± π/6 (60° ~ 120°) | rad |
| 动作缩放系数 | — | 0.003 | km/s |

### 坐标系

本地垂直/本地水平（LVLH）坐标系：原点位于目标航天器质心，x 轴沿径向向外，z 轴垂直于轨道平面（沿角动量方向），y 轴完成右手系（沿轨/切向）。所有位置和速度均为追踪航天器相对于目标在该系下的分量。

### 参与者 / 设备 / 时间

- **仿真平台**：Python（PyTorch + NumPy + SciPy）进行 PPO 训练，MATLAB（R2025a 及以上）进行批量后处理。
- **数据生成时间**：见末尾版本与生成日期。

## 数据结构详细说明

### 通用格式说明

- 每个 CSV 文件首行为列名（单列），之后每行为一个样本。
- 每行采用**嵌套列表字符串**格式：外层列表包含 10 个内层列表（对应 10 个时间步），每个内层列表包含 N×6 个浮点数（N 个目标 × 6 维状态），以 `%.4f` 格式保留 4 位小数，逗号后一个空格分隔。
- **重要**：不同来源（Capture2_new2 / Capture3_new2 / Capture4_new2）的样本中，N 值不同，因此每行内层列表的元素个数也不同（见下表）。

### X_now.csv

| 属性 | 说明 |
|------|------|
| 列名 | `X_now` |
| 物理意义 | 追踪航天器相对于目标的近期相对运动状态（观测窗口） |
| 数据结构 | 字符串格式嵌套列表 `[[step1], [step2], ..., [step10]]` |
| 时间步数 | 10 |
| 采样间隔 | h = 1 s |
| 总时间跨度 | 9 s（10 个点，间隔 1 s） |

**单步内层列表结构**（按目标依次排列）：

```
[x1, y1, z1, vx1, vy1, vz1,  x2, y2, z2, vx2, vy2, vz2,  ...,  xN, yN, zN, vxN, vyN, vzN]
```

| 分量 | 物理意义 | 类型 | 单位 | 数量 |
|------|---------|------|------|------|
| xi | 目标 *i* 的 x 方向相对位置（径向） | float | km | N 个 |
| yi | 目标 *i* 的 y 方向相对位置（切向） | float | km | N 个 |
| zi | 目标 *i* 的 z 方向相对位置（法向） | float | km | N 个 |
| vxi | 目标 *i* 的 x 方向相对速度 | float | km/s | N 个 |
| vyi | 目标 *i* 的 y 方向相对速度 | float | km/s | N 个 |
| vzi | 目标 *i* 的 z 方向相对速度 | float | km/s | N 个 |

> 每行共有 10 个内层列表，每个内层列表含 N×6 个 float。N 由样本来源决定（2 / 3 / 4）。

### X_next.csv

| 属性 | 说明 |
|------|------|
| 列名 | `X_next` |
| 物理意义 | 紧接 `X_now` 之后的下 10 个时间步的相对运动状态（预测目标窗口） |
| 数据结构 | 与 X_now 一致 |
| 时间步数 | 10 |
| 采样间隔 | h = 1 s（原始仿真部分）或 T = 60 s（CW 外推补齐部分） |
| 总时间跨度 | 取决于数据来源（混合 h 和 T 步长） |

> `X_next` 是截取/补齐后序列的第 11~20 步，**不是**从 X_now 终点独立递推生成的。在意图识别中，X_now→X_next 的映射关系可用于训练轨迹预测模型，也可以作为"真实演化"与独立 CW 递推预测之间的基准对比。

**与 `Dataset/X_recurrence.csv` 的关键区别**：

- `X_recurrence`：以 X_real 终点的状态为初值，用 CW 状态转移矩阵 Φ(T)（T=60 s）递推 10 步。**所有 10 步均为 CW 线性递推值**，步长均为 60 s。
- `X_next`：直接从截取/补齐后的序列取后半段。**大部分数据点来自真实仿真**（h=1 s 步长），仅在原始轨迹长度不足 `2 × real_length` 时，尾部用 Φ(T) 外推补齐（T=60 s 步长）。

## 行对应关系

每个样本是一个"批次"，由 MATLAB 脚本每次从 `Capture_T60h1/` 中随机抽取 N 个（2 或 3 或 4）有效的 `.mat` 轨迹文件，截取每个目标的最后一段轨迹进行处理，生成一行 CSV 数据。

**在同一个 Capture 源文件夹内**（以 Capture3_new2 为例）：

- 第 *k* 行 `X_now` 与第 *k* 行 `X_next` 取自同一批次的 3 个目标文件的同一段轨迹。
- 两个文件的行数严格相等（在合并之前已由 MATLAB 脚本保证）。

**经过 `merge_csv_new2.py` 纵向拼接之后**：

- 拼接顺序：Capture2_new2 样本 → Capture3_new2 样本 → Capture4_new2 样本。
- 行对应关系依然保持：两个 CSV 文件的行数相等，第 *i* 行 `X_now` 对应第 *i* 行 `X_next`。

> **严禁操作**：不得对两个 CSV 中的任何一个单独进行排序或随机打乱。若需划分训练/测试集，必须对两个文件同步进行（以相同随机种子对所有行索引做一致的分割或排列）。

## 数据预处理

以下列出从原始 `.mat` 轨迹数据到最终 CSV 文件的完整预处理流程：

### 1. 轨迹筛选（Python / MATLAB）

- **成功过滤**：仅保留 PPO 训练中成功拦截（相对距离 ≤ 5 km）的回合轨迹。
- **长度过滤**：仅保留 `length(t) > 10` 的轨迹（去除了时间步过短、无法截取有效片段的轨迹）。
- 数据存入 `Capture_T60h1/` 供 MATLAB 处理（共约 23,690 个 .mat 文件）。

### 2. 随机分批（MATLAB）

- 将所有 `.mat` 文件随机排列。
- 每次顺序取 N 个有效文件作为一批（N = 2 / 3 / 4，取决于脚本版本）。
- 不满足条件（`t` 不存在或 ≤ 10）的文件自动跳过。

### 3. 轨迹截取与外推（MATLAB）

- 对每个目标，截取轨迹的**尾部**：从 `end - mod(length_t, real_length) - (real_length - 1)` 到 `end`（确保能取到 `2 × real_length` 个点）。
- 若 `length_t mod real_length ≠ 0`（轨迹长度不是 10 的整数倍），用 CW 状态转移矩阵 Φ(T)（T=60 s）从截取段末尾向外推补齐至 2×real_length 步。

### 4. 序列切分（MATLAB）

- **X_now**：取截取/补齐后序列的**前 10 个点**（第 1~10 步）。
- **X_next**：取同一序列的**后 10 个点**（第 11~20 步）。

> 与主数据集不同，这里**不需要**再从 X_now 终点出发做 CW 递推。X_next 直接取自同一段截取序列。

### 5. CSV 输出（MATLAB）

- 数值以 `%.4f` 精度格式化为字符串，嵌套列表写入 CSV。
- **未进行归一化、标准化、滤波、缺失值填补**。这些预处理步骤留给下游神经网络训练流水线根据具体需求自行实施。

### 6. CSV 拼接（Python `merge_csv_new2.py`）

- 将 Capture2_new2 / Capture3_new2 / Capture4_new2 的同名 CSV 按序纵向拼接。
- 仅保留一份表头。
- 未做去重、排序或任何数据变更。

## 使用注意事项

1. **对齐要求**：两个 CSV 文件必须始终保持相同行序。任何划分、采样、数据增强操作需要同时对两个文件施加一致的索引变换。
2. **不可打乱**：两个文件是"输入-目标"的配对关系。单独对任一文件排序将导致第 *i* 行输入与第 *i* 行目标之间的对应关系永久错乱。
3. **拼接顺序**：Dataset_new2 中样本的顺序为 Capture2_new2 → Capture3_new2 → Capture4_new2。在切分训练/测试集时建议随机打乱（两个文件同步），而非按原始顺序切分。
4. **变长样本**：不同行的内层列表元素数不同（N×6 分别为 12 / 18 / 24），这对应不同的目标数量 N。神经网络模型需要处理变长输入，或按 N 分组训练。
5. **混合步长**：`X_next` 的后部可能包含 CW 外推点（步长 T=60 s），这与 `X_now`（全为 h=1 s 步长）的物理时间尺度不一致。如果下游任务对时间一致性有严格要求，需要考虑只使用原始长度充足的样本，或对此做适配处理。
6. **未归一化**：数据未经过任何归一化或标准化。位置量级约 0~200 km，速度量级约 0~0.1 km/s。训练神经网络时**强烈建议**按特征维度进行 z-score 标准化或 Min-Max 归一化。
7. **无缺失值**：所有样本均为有效轨迹，不包含 NaN 或缺失值。
8. **数据解析**：读取 CSV 后，每行数据为字符串，需要用 `ast.literal_eval` 或 `json.loads` 解析为嵌套列表（注意 Python 中解析后为 float 类型而非字符串）。
9. **CW 模型假设**：CW 外推基于 CW 方程线性化假设（圆轨道、近距离、无摄动），与真实非线性动力学存在模型误差。`X_next` 中外推部分的数据保真度低于真实仿真部分。
10. **与主数据集的关系**：本数据集的 `X_now` 与主数据集 `X_real` 在数学上等价（同一段截取序列的前 10 步）；但 `X_next` 与 `X_recurrence` 完全不同——前者是截取序列的直接延续，后者是独立 CW 递推的结果。两个数据集可用于互补分析：`Dataset` 侧重 CW 递推误差特征和几何标签，`Dataset_new2` 侧重纯轨迹预测。
